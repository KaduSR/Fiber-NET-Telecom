Guia de Implementação: Mobile & API

Este guia detalha os comandos de terminal necessários para inicializar os ambientes Mobile e Backend do projeto Fiber-NET-Telecom.

1. Inicializar o Projeto React Native (mobile/)

Escolha uma das opções abaixo para criar o aplicativo.

Opção 1: React Native CLI (Recomendado)

Esta opção oferece controle total sobre os módulos nativos.

Navegue até a pasta:

cd Fiber-NET-Telecom/mobile

Inicialize o projeto (sem instalar dependências automaticamente):

npx react-native init MyApp --skip-install

Mova os arquivos para a raiz da pasta mobile/ e limpe a pasta temporária:
(Execute um comando por vez ou copie o bloco para um script)

mv MyApp/* .
mv MyApp/.babelrc .
mv MyApp/.eslintrc.js .
mv MyApp/.gitattributes .
mv MyApp/.prettierrc.js .
mv MyApp/.watchmanconfig .
mv MyApp/App.js .
mv MyApp/index.js .
mv MyApp/metro.config.js .
mv MyApp/package.json .
mv MyApp/tsconfig.json . # Apenas se gerou TypeScript
rmdir MyApp

Instale as dependências:

npm install

Rodar o projeto:

npm run android

# ou

npm run ios

Opção 2: Expo CLI

Melhor para prototipagem rápida.

Crie o projeto na pasta atual:

cd Fiber-NET-Telecom/mobile
npx create-expo-app .

Rodar o projeto:

npm start

2. Desenvolver a API (api/)

Configuração do ambiente Node.js.

Navegue até a pasta e inicie o projeto:

cd ../api
npm init -y

Instale as dependências:

npm install express cors dotenv
npm install --save-dev nodemon

Criação dos Arquivos:

Utilize o código organizado no arquivo api_organization.md para criar o index.js.

Crie o arquivo .env na raiz de api/:

PORT=3001

Inicie a API:

npm run dev

3. Integração (Frontend <-> Backend)

Para conectar o mobile ou web à API:

Web: Aponte as requisições (axios/fetch) para <http://localhost:3001>.

Mobile (Emulador/Físico): O localhost do celular não é o mesmo do computador.

Descubra seu IP local (ex: ipconfig no Windows ou ifconfig no Linux/Mac).

Use o IP na URL: <http://192.168.X.X:3001>.
